2. Technical Description

Home Page & Content

HTML Structure

  1. <header>: Contains the logo and navigation links
    • <img>: Used to display the site logo
    • <nav>: Contains navigation links to other pages (ex:- Home, Sitemap, User Profile)
    • Purpose: To create a cohesive navigation system across the site
  2. <section class="welcome">
    • Contains the welcome message and an image to visually enhance the introduction to the site
    • <h1>: Displays the main heading (Life on Land Community), creating the title for the page
    • <p>: Provides a brief description of the site’s goals
    • Purpose: The welcome section introduces visitors to the core values of the website

CSS Styling

  1. Flexbox Layout: Used throughout the page to structure and align items, particularly in the navbar and .welcome section
    • .navbar: Positioned using display: flex to place site details and navigation in a row, with space between them
    • .site-links .active: The "active" class is used to highlight the current page link in the navigation bar
  2. Responsive Design:
    • The use of vw (viewport width) and vh (viewport height) ensures the page scales according to the screen size. However, for smaller screens, using rem or em might be better for readability

Sitemap Page

HTML Structure

  1. <section class="sitemap">
    • Displays a map-like structure of the site to help users navigate between different pages
    • <svg>: The site’s layout is represented using an SVG map, which is visually appealing and scalable

CSS Styling

  1. Flexbox & Grid Layouts
    • display: flex is used to align the content properly, while display: grid can be applied to ensure better organization and control over the elements in the map
  2. Hover Effects
    • rect:hover: When the user hovers over an area of the sitemap, the background color changes to improve user interaction
  3. accessibility Considerations
    • <a> links: Ensure the sitemap links are accessible, with proper focus styles to highlight the link when the user navigates using the keyboard

User Profile Page

HTML Structure

  1. <section class="profile">
    • Contains the user’s profile, including their image, skills, and goals
    • <img>: Displays the user’s profile picture
    • <h2> and <p>: These elements are used to display the user's personal information, such as name, skills, and goals

CSS Styling

  1. Flexbox Layout: Used to position the profile image and the other content side-by-side
    • .user: Uses flexbox to align the image and the user's details horizontally
  2. Responsiveness
    • Media Queries: Adjust the layout for smaller screen sizes by stacking elements vertically
  3. Styling for accessibility
    • Contrast: Proper contrast between background and text to ensure readability for users with visual impairments
    • Alt Text for Images: Proper alt text for all images to ensure the content is accessible to screen readers

5. Challenges and Lessons Learned

Challenges

  1. Responsive Design: Ensuring the layout worked on all screen sizes was challenging. The use of vw and vh units worked well for larger screens, but smaller screens required adjustments using media queries
  2. accessibility: Ensuring proper alt text and making the site accessible for screen readers was a learning process. I had to make sure all images had alt attributes and that all interactive elements were focusable and accessible

Lessons Learned

  1. I learned the importance of thorough testing on various devices to ensure responsiveness
  2. I gained experience in improving accessibility by using semantic HTML and testing keyboard navigation

6. Accessibility

Specific accessibility Considerations

  1. Alt Text: All images have descriptive alt text for screen readers
  2. Keyboard Navigation: Ensured that all links and interactive elements are keyboard-navigable
  3. Color Contrast: Checked that text and background colors meet WCAG accessibility standards for readability

7. Compliance

The pages I’ve created comply with Janet Regulations by following best practices for:

  1. Semantic HTML: Ensures the proper use of elements like <header>, <footer>, <section>, and <nav>
  2. WCAG compliance: Implemented accessibility features like alt text, keyboard navigation, and color contrast to meet WCAG standards
